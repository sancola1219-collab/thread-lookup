# parseFloat 對中文混合字串會偷取開頭數字，搜尋要先驗純數字

一行摘要：`parseFloat("4分") === 4`，導致搜尋「4分」時外徑 3～5mm 的 M3/M4/M5 全被誤判命中。

**為什麼**：原本設計「輸入數字 → 比對外徑 mm ±1.2」方便用卡尺量完直接查。但 parseFloat 遇到「4分」「3吋」這種字串不會回 NaN，而是取開頭的數字，讓文字查詢誤觸數值比對。

**解法**（已套用在 applyFilter()）：

```js
const qn = /^[\d.]+$/.test(q) ? parseFloat(q) : NaN; // 純數字才啟用外徑比對
```
