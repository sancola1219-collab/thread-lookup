# 旋轉的 3D 螺桿讀不出牙角與牙距，尺寸要用「軸向剖面詳圖」

一行摘要：使用者要「在圖上看到所有尺寸與角度」，第一版做了可旋轉 3D 螺牙——但 3D 上根本量不到牙角(55°/60°)與牙距，還被誤會成「平滑圓桶」。正解是加一張全尺寸標註的 2D 剖面詳圖當主角。

**為什麼**：
- 牙角是「兩側牙腹的夾角」，只有在通過軸心的縱向剖面上才是真實角度；3D 透視＋螺旋＋轉動會讓它失真，量不到。
- 牙距同理，要沿軸線量相鄰牙峰，旋轉視角對不齊。
- 大徑(21mm)和牙深(1.16mm)差 18 倍，同一張 3D 上一個看得到一個看不到。

**解法**（已套用）：overlay 主角改成 `buildProfileSVG()` 產的 2D 剖面詳圖：
- 主視圖上下鏡射縱剖，垂直距離＝真直徑（大徑/底徑用垂直尺寸線一次量到底）。
- 另畫「牙型放大詳圖」用**真實牙角**畫 2 顆牙，角度弧才誠實——PT 55° 與 NPT 60° 一眼可分。
- 徑向可放大示意，但每個標註**數字都是真值**，並在圖上明標「放大示意」。
- 3D 降為輔助「立體長相」，明標「精確尺寸見剖面詳圖」。

**驗證法**：不用看像素，直接量幾何——抓詳圖裡兩條牙腹延伸線的頂點，算夾角，斷言 ≈ 55/60°（實測 55.06°、59.92°，通過）。

**牙紋看得見的關鍵**（3D 那側）：加「假環境光遮蔽 AO」——牙峰亮、牙底暗，依局部半徑計算、**與視角無關**；配合牙深視覺放大＋圓周分段提高。只靠方向光＋背面剔除會把牙腹明暗抹平成光滑圓桶。
